## 1. MCP Server Virtual Environment Setup and Configuration

**Summary:** The MCP server project had both .venv and venv directories causing confusion, and the server wasn't configured to run with Claude Code despite being manually executable. ### Technical Approach
- **Package manager:** Used uv instead of pip for faster dependency resolution and installation
- **Virtual environment:** uv creates .venv/ by default (hidden directory convention)
- **MCP configuration:** Claude Code reads .mcp.json from project directories, requires mcpServers wrapper
- **Server approval:** Must add server names to enabledMcpjsonServers in ~/.claude/settings.json
- **Integration:** Server runs via stdio transport, Claude Code spawns process on demand

### Key Learnings
- **uv creates .venv by default** (not venv) - hidden directory keeps project root cleaner
- **.mcp.json requires mcpServers wrapper** - direct server objects at top level cause parse errors
- **Claude Code doesn't read mcpServers in settings.json** - must use enabledMcpjsonServers for .mcp.json approval
- **MCP servers must be approved in settings** - adding to .mcp.json alone is insufficient
- **Restart Claude Code required** - configuration changes only take effect after restart
