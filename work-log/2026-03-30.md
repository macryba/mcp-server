## 1. MCP Server Troubleshooting and Migration to Python FastMCP

**Summary:** Existing SearXNG-based MCP server was experiencing persistent bot detection issues (403 Forbidden) despite implementing all recommended fixes from official documentation including limiter.toml configuration, nginx reverse proxy with proper headers, and trusted_proxies settings. Made decision to abandon Docker/SearXNG approach and migrate to lightweight Python FastMCP server using Wikipedia API directly. Goal was to create reliable, CAPTCHA-free search for Polish history research.

- **SearXNG bot detection is extremely difficult to bypass** in Docker/LAN setups despite official documentation
- **limiter.toml configuration alone insufficient** - requires proper reverse proxy headers
- **Direct Wikipedia API is more reliable** for focused use cases than general metasearch
- **FastMCP provides lighter, simpler architecture** than Docker-based MCP servers
- **Permission issues in Docker** (UID 977:977) required repeated chown commands, solved with user directive
